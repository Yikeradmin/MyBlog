索引

2024-06-05
Uncategorized
1.1k words
  • Mysql

作用

  • 提高查询速度
  • 确保数据唯一性
  • 可以加快表之间的连接，实现表之间的参照完整性
  • 使分组和排序子句进行数据检索时，可以显著减少分组和排序的时间
  • 全文索引字段进行搜索优化

分类

  1. 主键索引 PRIMARY
    唯一索引、不能重复、不允许为空
  2. 唯一索引 UNIQUE
    唯一索引、值不能重复、可以为空
  3. 常规索引 INDEX
    常规索引
  4. 全文索引 FULLTEST
    只能用于字符串类型的字段

主键索引 PRIMARY KEY

主键：某一个属性组能够唯一标识一条记录

特点：最常见的索引类型；确保数据的唯一性；确定数据记录在数据库中的位置

创建表时指定索引

创建了表之后添加索引

唯一索引 UNIQUE

作用：避免同一个表中某个数据列的值重复

与主键的区别：主键只能有一个，但是唯一索引可以有多个

例如：指定age和name字段的值不能重复

创建表之后添加索引

常规索引 INDEX

作用：快速定位到特定数据

注意：index 和 key 关键字都可以设置常规索引；应加在查询条件的字段中；不宜添加太多，影响数据的插入、修改和删除的速度

全文索引

作用：快速定位特定的数据

注意：只能用于MylSAM类型的数据表；只能用于CHAR、VARCHAR、TEXT数据列类型；适用于大类型数据集

写在最后

索引不是越多越好

不要经常变动索引的数据和加索引

小数据量的表建议不要加索引

索引一般加在查找条件的字上
